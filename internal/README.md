# Internal Documentation

This folder contains internal documentation for Radia maintainers and contributors.

## Folder Structure

### design/
Architecture decisions, design documents, and implementation proposals.

**Contents:**
- `BATCH_EVALUATION_IMPLEMENTATION_COMPLETE.md` - Batch evaluation feature completion report
- `BATCH_EVALUATION_PROPOSAL.md` - Original proposal for batch field evaluation
- `BATCH_IMPLEMENTATION_PLAN.md` - Implementation plan for batch evaluation
- `HMATRIX_CACHE_IMPLEMENTATION.md` - H-matrix cache implementation details
- `PREPARECACHE_OPTIMIZATION_PROPOSAL.md` - PrepareCache performance optimization proposal
- `PYTHON_CACHED_FIELD_SOLUTION.md` - Pure Python cached field solution design

### analysis/
Performance analysis, bottleneck investigations, and optimization reports.

**Contents:**
- `BATCH_EVALUATION_BOTTLENECK_FINAL_ANALYSIS.md` - Final analysis of batch evaluation bottlenecks

## Purpose

These documents are for:
- **Maintainers**: Understanding design decisions and implementation details
- **Contributors**: Learning about past optimizations and avoiding duplicate work
- **Performance engineers**: Analyzing bottlenecks and planning optimizations

## Not for End Users

End-user documentation is in the [docs/](../docs/) folder:
- API references
- User guides
- Feature documentation
- Quick start guides
